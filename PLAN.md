# 작전 바이브 시뮬 - 소부대 통신 전술 훈련 시뮬레이터

## 프로젝트 개요
- **프로젝트 명**: Signal-Fog (작전 바이브 시뮬 / 무전 오류 훈련 시뮬레이터)
- **슬로건**: "통신이 살아야 작전이 산다 — 대대장님, 분대 이동 명령 하달!"
- **팀 구성**: 팀명 - LNG : 2인 팀 (상병 김응태 + 상병 장호영)
- **대회**: 2026 제15회 육군창업경진대회 예선 출품
- **형태**: 웹 기반 1인 턴제 탑뷰 2D 전술 시뮬레이션 (Phaser.js CDN, 로컬 단독 실행)
- **목표**: 군 소부대 작전 훈련 중 발생하는 **무전기 오류·통신 왜곡·판단 미스**를 반복 학습 가능한 시뮬레이션으로 해결 → 실전 오류율 감소 + 장병 훈련 효율화

## 문제 정의 (군 생활 Pain Point)
소부대(분대·소대) 작전 및 KCTC 훈련 시 가장 빈번한 실패 원인 중 하나:
- 무전기 통신 지연 / 노이즈 / 오청 → 수신호·판단 오류 발생
- 실제 훈련 전 미리 연습할 방법 부족 → 실전에서 반복 실수
- 결과: 작전 실패, 시간 지연, 안전 문제, 전투력 저하

→ 모든 육군 장병(특히 전방·전투병과)이 공감하는 **실전 통신·판단 고충**
(역대 수상작 중 모션인식 장갑 '플랙사인'처럼 통신/수신호 관련 아이템이 최우수상 받은 사례 있음)

## 솔루션: 작전 바이브 시뮬레이터 (예선 MVP 기준)

### 핵심 플레이 구조
- **대대장 시점**: 플레이어 1인이 아군 분대 3~5개를 직접 조작·지휘
- **AI 적군**: Gemini API(무료 플랜) 기반 적 지휘관 — LLM이 현재 전황을 받아 적 유닛 행동을 결정
- **8×8 그리드 맵**: 예선 프로토타입 기준, 지형 3~4종 배치
- **핵심 메커니즘 — 오청(Mishear)**: 플레이어가 분대에 내린 명령이 통신 품질에 따라 텍스트 변형되어 전달됨 (예: "3번 타일로 이동" → "5번 타일로 이동")
- **포그 오브 워**: 시야 밖 적 위치 미표시, 정보 불완전성 재현
- **턴 행동 제한**: 유닛당 이동 + 행동 1~2회 → 실제 작전 리듬감

### 이중용도 (군 + 민간)
- **군 내**: 소부대 훈련 보조 도구 (KCTC 전 통신 오류 반복 연습 → 오류율 20~30% 감소 기대)
- **민간 확장**: Airsoft/서바이벌 클럽 전술 훈련, ROTC·예비군 교육 툴, 기업 팀빌딩·리더십 시뮬, e스포츠 전략 게임 시장 진입

### 개발 철학
사지방 제약 100% 고려 — 설치 0, GitHub Pages 배포, 브라우저 즉시 실행. Firebase·멀티플레이 없이 로컬 단독 실행으로 네트워크 리스크 최소화.

## 기술 스택 (예선 MVP)
- **게임 엔진**: Phaser.js 3.90.0 (CDN)
- **적 AI**: Gemini API (Google AI Studio 무료 플랜, Gemini Flash) — 외부 fetch 호출
- **배포**: GitHub Pages (무료, 사지방 접근 용이)
- **협업**: GitHub private repo (2인 무제한 collaborators)
- **사운드**: Howler.js (CDN) — 무전기 효과음·노이즈
- **에셋**: Kenney.nl / OpenGameArt.org (CC0 무료)

> TensorFlow.js 및 Firebase는 예선 MVP에서 제외. 구현 복잡도 대비 효용이 낮고, 사지방 환경에서 외부 의존성이 많을수록 리스크가 커지기 때문.

## 개발 로드맵

### Phase 1 — 예선 MVP (~ 4월 10일)
- 8×8 그리드 맵 + 지형 3~4종
- 대대장 시점 아군 분대 조작 (이동·사격)
- Gemini API 기반 AI 적군 행동
- 오청(Mishear) 1종 통신 오류 구현
- 포그 오브 워 기본
- 유튜브 데모 영상 촬영 (1~2분)

### Phase 2 — 본선 고도화 (본선 진출 시)
- 맵 확장 (20×20 헥스 그리드 전환)
- 통신 오류 5종 전체 구현 (오청·지연·두절·도청·위장삽입)
- 야간 페이즈 + 생존 스탯 (피로·배고픔·수면) 완전 구현
- 멀티플레이 도입 (Firebase Realtime DB 또는 WebRTC)
- AAR 시스템 (히트맵·리플레이)
- 보급 시스템·인수인계 체계 구현
- 모바일 터치 UI 최적화
- Gemini AI 난이도 자동 조절 (전황 복잡도에 따라 프롬프트 변화)

### Phase 3 — 전역 후 창업 연계
- 창업사관학교 / TIPS / 국방 스타트업 챌린지 연계
- B2G: 국방부·육군 교육훈련용 납품 목표
- B2C: Freemium 모델 (기본 무료 + 프리미엄 미션·지도)
- B2B: 기업 팀 전략 훈련 툴 판매

## 사업화 계획 (대회 신청서용)
- **현재 단계**: 웹 프로토타입 (아이디어 + 작동 데모)
- **시장 잠재력**: 글로벌 밀리터리 시뮬레이션 시장 + e스포츠 전략 게임 시장 (수백억 규모)
- **BM 요약**: 군 복무 중 겪은 실전 통신 오류를 재미있는 워게임으로 해결 → 링티처럼 군 생활 고충에서 출발한 창업 서사

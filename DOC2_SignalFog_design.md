## 2. 핵심 컨셉

> "통신이 살아있어야 작전이 살아있다 — 대대장님, 분대 이동 명령 하달!"

KCTC 훈련에서 가장 빈번한 실패 원인인 **통신 오류·판단 미스**를 게임의 핵심 메커니즘으로 설계. 플레이어는 대대장 시점에서 아군 분대를 지휘하고, Gemini API가 구동하는 AI 적 지휘관과 전술 대결을 벌인다.

단순 전투 게임이 아니라 **명령→전달→오청→실행 오류** 전 과정을 체험하는 훈련 시뮬레이터.

---

## 3. 플레이어 구조 — 대대장 시점

### 3-1. 설계 원칙

플레이어 1인이 아군 분대 3~5개를 직접 조작한다. 각 분대에 명령을 내리면 통신 품질에 따라 오청이 발생, 명령이 의도와 다르게 전달될 수 있다. 플레이어는 오청을 인지·보정하며 전술을 조율해야 한다.

```
[플레이어 (대대장 시점)]
   ├─ 1분대 (소총수 3명)
   ├─ 2분대 (소총수 3명)
   ├─ 3분대 (기관총수 + 소총수)
   ├─ (선택) 4분대 — 박격포 운용
   └─ (선택) 5분대 — 의무·보급

[AI 적군 (Gemini API)]
   ├─ 적 분대 3~5개
   └─ 매 턴 전황 데이터를 받아 이동·공격 행동 결정
```

### 3-2. 턴 구조

```
[명령 입력 페이즈]
   └─ 플레이어가 각 분대에 행동 입력 (이동 타일 클릭 또는 명령 텍스트)
   └─ 통신 품질 판정 → 오청 발생 시 명령 텍스트 변형

[실행 페이즈]
   └─ 아군 분대 행동 순차 실행
   └─ Gemini AI 적군 행동 결정 및 실행
   └─ 교전 판정

[결과 확인]
   └─ 피해 현황, 통신 오류 발생 여부 표시
```

## 4. 맵 시스템

### 4-1. 예선 MVP 맵 규격

| 구분 | 예선 MVP | 본선 확장 |
|------|---------|---------|
| 맵 크기 | 8×8 정사각형 그리드 | 20×20 헥스 그리드 |
| 지형 종류 | 3~4종 | 8종 전체 |
| 스크롤 | 고정 (전체 화면) | 드래그 스크롤 |

### 4-2. 예선 MVP 지형 3~4종

| 타일 | 이동 비용 | 통신 효과 | 엄폐 |
|------|---------|---------|------|
| 개활지 | 1 | 정상 | 없음 |
| 수풀·숲 | 2 | 정상 | +10% |
| 계곡·저지대 | 2 | **두절 (오청 확률 대폭 증가)** | 낮음 |
| 고지 | 2 | 통신 범위 +1 | 낮음 |

> 계곡 지형은 오청 핵심 메커니즘 시연 용도. 예선 데모에서 반드시 포함.

### 4-3. 본선 확장 지형 (8종 전체)

문서 2 v0.3 기준 지형 8종 그대로 유지. 헥스 그리드 전환과 함께 구현.

## 5. 핵심 메커니즘 — 오청(Mishear) 시스템

### 5-1. 예선 MVP 오청 1종

**발생 조건**: 통신 품질 70% 이하 (계곡 지형 진입, 무전기 배터리 소모 등)

**효과**: 플레이어가 입력한 명령 텍스트의 좌표·방향이 일부 변형되어 분대에 전달됨

```
예시 A — 좌표 변형:
  입력:    "3번 타일로 이동"
  전달:    "5번 타일로 이동"
  결과:    분대가 엉뚱한 타일로 이동

예시 B — 방향 변형:
  입력:    "북쪽 고지 점령"
  전달:    "북동쪽 고지 점령"
  결과:    분대가 측면 노출 지형으로 이동

예시 C — 명령 누락:
  입력:    "2번 분대 엄폐 후 사격"
  전달:    "2번 분대 사격"
  결과:    엄폐 없이 개활지에서 사격 → 피해 증가
```

**시각 표현**: 오청 발생 시 채팅창에 글리치 텍스트 효과 + 경고음

### 5-2. 본선 확장 통신 오류 5종

| 오류 유형 | 예선 | 본선 |
|---------|------|------|
| 오청 (Mishear) | ✅ 구현 | 고도화 |
| 지연 (Delay) | ❌ | ✅ |
| 두절 (Blackout) | ❌ | ✅ |
| 도청 (Intercept) | ❌ | ✅ |
| 위장 삽입 (Spoof) | ❌ | ✅ |

## 6. AI 적군 — Gemini API

### 6-1. 설계 원칙

적 유닛의 이동·공격 행동을 Gemini API(무료 플랜, Gemini Flash 모델)로 결정한다. 랜덤 AI가 아니라 LLM이 현재 전황을 읽고 전술적 판단을 내린다.

### 6-2. 동작 방식

```
[매 턴 실행 페이즈]
   └─ 현재 맵 상태 직렬화
       ├─ 아군 분대 위치·잔여 병력·통신 상태
       ├─ 적 분대 위치·잔여 병력
       └─ 지형 정보

   └─ Gemini API 호출 (fetch)
       └─ 프롬프트: "당신은 KCTC 대항군 지휘관입니다.
                    현재 전황: [맵 상태 JSON]
                    적(아군)의 통신이 [X]% 품질입니다.
                    다음 턴 각 적 유닛의 행동을 JSON으로 반환하세요."

   └─ 응답 파싱 → 적 유닛 행동 실행
       ├─ 이동: 타일 좌표
       └─ 공격: 대상 유닛 ID
```

### 6-3. 리스크 및 대응

| 리스크 | 대응 |
|--------|------|
| 사지방 외부 API 차단 가능성 | 사전 테스트 필수. 차단 시 랜덤 이동 폴백 로직으로 전환 |
| API 응답 지연 | 로딩 표시 + 타임아웃 3초 설정, 초과 시 폴백 |
| 무료 플랜 요청 한도 | 턴당 1회 호출 → 데모 시연 범위에서 한도 초과 없음 |
| 비정형 응답 | JSON 파싱 실패 시 폴백 행동 (제자리 대기) |

### 6-4. 심사 포인트

단순 랜덤 AI가 아닌 **LLM 기반 전술 판단 AI 탑재**라는 점이 기술 차별화 포인트. 본선 질의응답 예상 답변: "Gemini가 현재 맵 상태를 텍스트로 받아 적의 다음 행동을 JSON으로 반환합니다. 이를 통해 상황에 따라 포위·회피·집중 공격 등 전술적 행동을 보입니다."

## 7. 교전 판정 시스템 (예선 MVP 간소화)

### 7-1. 예선 MVP 판정 흐름

```
사격 선언
   └─ 사거리 체크
   └─ 명중 판정 (기본 명중률 + 지형 보정)
   └─ 명중 → 즉사 또는 부상
   └─ 빗나감 → 소음 발생 (위치 노출 여부)
```

### 7-2. 예선 MVP 무기 (소총 1종)

| 무기 | 사거리 | 기본 명중률 | 피해 |
|------|--------|-----------|------|
| 소총 | 4타일 | 60% | 즉사·부상 |

> 예선은 소총 단일 구현. 기관총·박격포·포병은 본선 확장.

### 7-3. 본선 확장 무기 체계

문서 2 v0.3 §8-2 무기별 스펙 그대로 유지.
